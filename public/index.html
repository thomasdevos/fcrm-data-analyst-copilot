Executive Summary

The Challenge

Financial Crime and Fraud Prevention (FCFP) L1 analysts spend significant manual effort gathering and reviewing data from multiple systems, primarily FCRM, just to perform initial triage of alerts. This repetitive, time-consuming process slows down case resolution and creates operational bottlenecks.

The Solution

A GenAI-powered AI Assistant that provides two core functions: Proactive Summarization of case data when an analyst enters a Case ID, and Interactive Q&A allowing analysts to interrogate case details through natural-language conversation. The Copilot accelerates information gathering and investigation without performing triage or decision-making.

Expected Impact

  • 30-50% reduction in information gathering & investigation time
  • >95% factual accuracy in summaries and Q&A responses
  • Interactive AI Assistant for natural-language case interrogation
  • UAT-ready application on mock data, path to live integration

Co-Innovation Project

This is a pragmatic, deliverable-focused MVP developed as a co-innovation project between HCLTech and ING. The solution demonstrates secure GenAI deployment in compliance operations, delivering a UAT-ready Copilot application that aids analysts without interfering with existing systems like SATM.

MVP Scope Definition

‚úÖ In Scope for MVP (Target for UAT)

Functionality Included:

  • Standalone web application with Case ID input
  • Proactive Summarization: Automatically generate natural-language summary of key case facts
  • Interactive Q&A: AI Assistant chat interface to interrogate case data (e.g., "What was the total value of outbound transactions?")
  • Mock Data Foundation: Built and tested using high-fidelity mock dataset based on FCRM/CDT data model
  • Secure GCP deployment with VPC Service Controls and IAM
  • Audit logging and explainability layer for transparency
  • UAT-ready application with path to live data integration (Release 2)

‚ùå Out of Scope for MVP

Explicitly Excluded:

  • Any decision-making, triage, recommendations, or proposals (e.g., "Close" or "Escalate")
  • Automated report generation for case closure
  • Direct "write-back" integration to FCRM (e.g., no auto-closing cases)
  • Live data integration from FCRM API, DAP, Quantexa, or TMA (planned for Release 2)
  • Automated aggregation or creation of new alerts
  • Autonomous/agentic AI behavior (ING policy restricts this pending broader approval)

Note: The Copilot strictly aids analysts - it does not interfere with existing systems like SATM

Risks, Mitigations & Key Decisions

This section outlines critical risks related to data privacy, regulatory compliance, and LLM strategy, along with proposed mitigation approaches and decisions required from ING stakeholders.

üî¥ CRITICAL

Data Privacy & PII Transmission Risk

Risk Description

FCRM case data contains highly sensitive PII and financial information including customer names, addresses, tax IDs, account numbers, and transaction details. Using external LLM APIs (e.g., Google Gemini, Azure OpenAI) transmits this data outside ING's control, creating:

  • GDPR Article 28 compliance risk - Third-party data processors require Data Processing Agreements (DPA)
  • ECB/DNB regulatory risk - May prohibit sending customer PII to external cloud APIs
  • NIS2 & DORA compliance - EU regulations for critical financial infrastructure
  • Data residency requirements - Data may need to remain within EU/NL jurisdiction
  • Potential data breach exposure - External vendor security incidents

Mitigation Approaches (Decision Required)

üèÜ Option 1: On-Premise LLM Deployment (RECOMMENDED for Production)

Deploy open-source LLM within ING infrastructure - No data leaves ING network

  • Models: Meta Llama 3.1 (8B/70B), Mistral 7B, Mixtral 8x7B
  • Deployment: ING's GCP GPU cluster or on-premise Kubernetes
  • Pros: 100% data sovereignty, GDPR compliant, no per-token costs, full control
  • Cons: GPU infrastructure costs (‚Ç¨10k-50k), ML Ops overhead, fine-tuning required
  • Timeline: 8-12 weeks additional setup for production deployment

Decision Point: ING to assess feasibility of GPU infrastructure provisioning and approve ML Ops resourcing for production phase.

Option 2: Google Vertex AI with Private Service Connect (Hybrid Approach)

Use Gemini via Google Cloud with EU data residency + private VPC

  • Deployment: Vertex AI in europe-west4 (Netherlands) with Private Service Connect
  • Data flow: Data stays in EU, transmitted over private network (no public internet)
  • Pros: Enterprise DPA available, VPC isolation, EU compliance, managed service
  • Cons: Still external processor, complex setup, vendor lock-in, compliance approval needed
  • Timeline: 4-6 weeks for VPC setup and security approval

Decision Point: Legal & Compliance to approve Google as data processor with signed DPA. Infrastructure team to provision Private Service Connect.

Option 3: Mock Data Only (MVP Phase - Current Approach)

Use external APIs (Gemini) ONLY with synthetic/mock data for MVP demo

  • Approach: Build and validate functionality using realistic but fictional mock cases
  • Transition: Switch to Option 1 or 2 before connecting to live FCRM data
  • Pros: Fast development, no compliance blockers for MVP, stakeholder validation
  • Cons: Cannot go to production without switching to compliant LLM approach
  • Timeline: Supports current 19-week MVP timeline

Decision Point: Approve use of Gemini API for MVP demo purposes only with commitment to migrate to Option 1/2 before live data integration.

Strategic Recommendation

Phase 1 (MVP): Proceed with Option 3 - Use Gemini API with mock data only to deliver UAT-ready prototype and validate business value.

Phase 2 (Production): Deploy Option 1 - On-premise Llama 3.1 70B for full data sovereignty and cost-effectiveness at scale.

Fallback: If GPU infrastructure unavailable, pursue Option 2 - Vertex AI with Private Service Connect as interim solution with strict DPA and audit controls.

üü† HIGH

LLM Vendor Strategy & Avoidance of Non-Strategic Platforms

Risk Description

ING may have strategic preferences or restrictions on LLM vendors. Azure OpenAI has been explicitly identified as not a preferred strategic approach for ING.

Implications

  • Vendor lock-in risk - Dependence on non-strategic vendor complicates long-term support
  • Contract limitations - ING may lack enterprise agreements with certain vendors
  • Compliance complexity - Non-approved vendors require additional legal/security reviews

Mitigation

This proposal excludes Azure OpenAI from consideration and focuses on:

  • Google Gemini (Vertex AI) - If ING has GCP enterprise agreement
  • Open-source models (Llama, Mistral) - Vendor-neutral, self-hosted approach

Decision Point: ING to confirm approved LLM vendors and strategic platform preferences (GCP vs. vendor-neutral open source).

üü† HIGH

Regulatory Compliance & Audit Trail

Risk Description

Financial crime investigations are highly auditable. Use of AI in case analysis requires:

  • Full audit trail of all AI queries and responses
  • Explainability - Ability to show how AI reached conclusions
  • Human oversight - AI cannot make triage decisions (already scoped correctly)
  • DPIA completion - Data Protection Impact Assessment required

Mitigation

  • All AI interactions logged to audit database with timestamps and user IDs
  • Source citations - AI responses reference specific data points (case IDs, transaction IDs)
  • Read-only access - AI cannot modify case data or make decisions
  • DPIA template - HCLTech to provide standard DPIA questionnaire for ING completion

Decision Point: ING Compliance/Legal to complete DPIA assessment and approve AI use case before UAT with real data.

üü° MEDIUM

LLM Accuracy & Hallucination Risk

Risk Description

LLMs can generate plausible but incorrect information ("hallucinations"), which could mislead analysts if AI summarizes or interprets case data incorrectly.

Mitigation

  • Grounding in source data - AI only uses provided case data, no external knowledge
  • Citation required - All claims must reference specific data fields
  • Confidence disclaimers - UI shows "AI-generated summary" warnings
  • UAT validation - Subject Matter Experts validate accuracy during testing
  • Success metric: >95% factual accuracy measured during UAT

Decision Point: Define acceptable error rate threshold (propose <5% hallucination rate) and UAT acceptance criteria.
